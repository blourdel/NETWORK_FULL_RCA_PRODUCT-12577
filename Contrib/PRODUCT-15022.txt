Short Abstract / Blogline 

Smoothen the handover of network-related process problems to the network team

Enrich the “events” section of the existing process “network Problems” with all insights belonging to the network area(s) serving the process(es)

Customer Zero 


EDE: application installed on  ACI fabrics in Linz or other main offices. 

Process-to-Process problem root cause analysis assistance.


Target Audience 

Application owner faced with the need to delegate the resolution of a process or process-to-process network problem

Network teams that receive requests for assistance from the Application owner in case of a network problem

CIOs concerned by the network infrastructure opacity, high MTTR, and tooling costs


WHY?


Mitigate the following pains. This VI does not offer a full solution, though.

Users

Application observability RCA ends in an empty “network problem”

Customers

•Inability to justify the cost of retraining the network team on a new tool (Dynatrace) with a poorer feature set and UX

Dynatrace

Do not deliver a true full stack Davis® AI, preventing Dynatrace from fulfilling its promises

Functional Requirements / Solution Journey



Enrich the existing process “network Problems” with network insights

•“network area”: a bag of network devices, topologically related

•Attach  a “network area” to the host  running the process(es)

•Display host-attached “network area” Alerts/Warning/Events


Network Problems 

Infra mode: network slowness

Infra mode: network packet drops

Full-stack mode: network slowdown

Full-stack mode: network error

The health alerts, warning signals, and custom alerts are selected based on the following method at the  process to process occurrence time: 

There is at least one router in the path between the two processes

Analyze the network flow event produced by the OneAgent: source and destination IP address

Identify all network areas to which the source and destination IP address belongs

Identify all the health alerts, warning signals, and custom alerts belonging to these network areas

There are only Layer 2 switches in the path between the two processes
